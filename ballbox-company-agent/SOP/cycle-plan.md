# Cycle Plan SOP

1. Scan `signals/`, `.inbox/`, `.tasks/`, and active project plans.
2. List opportunities and unresolved questions.
3. Rank by expected payoff, speed to learning, and dependency risk.
4. Pick one main effort.
5. Write hypotheses, outputs, risks, and budget into the next cycle plan file.
6. Remove work that does not clear the budget bar.
